Your AI agent is only as safe as the skills you install

The AI agent skill ecosystem is growing at breakneck speed — skills.sh already hosts 90K+ skills with zero curation, vetting, or sandboxing. It's npm in 2013 all over again, except the stakes are higher: skills have shell access, file system read/write, network permissions, and direct access to your prompts and secrets.

Every skill you install from an untrusted source is a potential backdoor into your development environment, CI/CD pipeline, or production infrastructure.

Get Started

Scan in seconds

Three commands cover the common workflows: ad-hoc scanning, CI integration, and allowlist management.

Scan a skill directory
# Scan a skill directory
$ skill-scan scan ~/.claude/skills/
CI/CD mode with SARIF output
# CI mode (exit code 1 on issues)
$ skill-scan scan . --ci --output sarif > results.sarif
Using an allowlist for false positive suppression
# Scan with allowlist
$ skill-scan scan ./skills --allowlist .skill-scan-allowlist.yaml
